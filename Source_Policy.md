# Hoax.ai Source Policy

## Purpose

Hoax.ai must be built as a source-disciplined reference asset.

The project cannot rely on random AI-generated claims, unsourced statistics, sensational examples, or unstable assertions.

Source quality is part of the asset value.

## Source Discipline

Every factual statement that supports a public reference page, standard, protocol, or tool explanation must be grounded in reliable source material.

Hoax.ai may use:

- official documentation;
- technical standards;
- academic research;
- public policy documents;
- reputable security research;
- platform trust-and-safety documentation;
- media verification references;
- cybersecurity reports;
- provenance and authenticity standards;
- legal or regulatory materials when carefully framed;
- and clearly identified primary sources.

## Source Classes

Sources should be classified by type:

- Primary standard;
- Official institutional source;
- Academic paper;
- Technical documentation;
- Security research;
- Legal or policy source;
- Platform policy;
- Industry report;
- Reference article;
- Public dataset.

## Prohibited Source Practices

Hoax.ai must not:

- cite weak sources as authority;
- build reference pages from unsupported AI output;
- use sensational news as category proof without context;
- rely on unverifiable social media claims;
- use individual accusations as examples in early public pages;
- or create pages around current controversies without strong governance.

## Evidence Examples

Early public examples should avoid naming real individuals, active political controversies, or unresolved accusations.

The first version should use abstract, generic, or synthetic examples to explain evidence posture.

This protects the asset from becoming a dispute platform before its governance layer is mature.

## Source Registry Direction

A future source registry should track:

- source title;
- source type;
- publisher;
- date;
- URL;
- topic covered;
- claim supported;
- reliability notes;
- and last reviewed date.

## Citation Principle

Sources must support the exact claim being made.

Do not use a source to imply more than it says.

## Governance Rule

**No reference authority without source discipline.**
