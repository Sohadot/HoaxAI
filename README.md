# Hoax.ai

**Hoax.ai is a governed evidence-risk intelligence system for the synthetic media age.**

Hoax.ai is not a truth machine, a political fact-checking site, or a generic AI detector.

It exists to examine the condition of evidence when images, voices, screenshots, documents, identities, sources, and contexts can be generated, altered, imitated, or simulated at scale.

The core question is not:

> Is this true?

The core question is:

> What is the condition of the evidence?

## Strategic Thesis

The old internet problem was misinformation: false or misleading claims moving through distribution systems.

The AI-era problem is deeper.

Artificial intelligence industrializes plausibility. It makes synthetic faces, cloned voices, fabricated documents, forged screenshots, invented quotes, imitated sources, and reconstructed contexts easier to produce and harder to inspect.

This creates **synthetic fragility**: the collapse of easy confidence in the evidence layer itself.

Hoax.ai reframes hoaxes from false content into failures of evidence structure.

## Operating Position

Hoax.ai classifies evidence posture, synthetic risk signals, source confidence, contextual gaps, provenance weaknesses, and deception patterns.

It does not issue careless truth verdicts.

It does not accuse individuals or institutions.

Evidence-risk classification is not a judgment about the person, institution, brand, organization, or event connected to the evidence.

It does not claim absolute detection.

It provides a governed framework for examining evidence before belief, action, publication, escalation, or institutional response.

## Sovereign Integrity

Hoax.ai does not ask for trust by assertion. It makes trust inspectable.

The [Self-Application Doctrine](SELF_APPLICATION.md) requires Hoax.ai to subject its own claims to the same evidence discipline it applies to the category. Foundational claims are recorded in the [Evidence Ledger](data/evidence-ledger.json) per [EVIDENCE_LEDGER_POLICY.md](EVIDENCE_LEDGER_POLICY.md). Reference-grade status is governed by the [Sovereign Reference Integrity Standard](SOVEREIGN_REFERENCE_INTEGRITY_STANDARD.md).

## First Build Direction

Hoax.ai will be developed as a GitHub-first sovereign digital asset.

The initial build will prioritize:

- category thesis;
- governance boundary;
- claim discipline;
- source policy;
- interface thesis;
- buyer logic;
- monetization boundary;
- roadmap;
- and later, a bounded evidence posture classifier.

The asset must become more than a website.

It must become category infrastructure: name, language, standard, reference layer, tool, governance, and strategic acquisition logic.

## Hosting and Infrastructure

The first public foundation will be built and published through GitHub.

Custom domain connection and Cloudflare nameserver routing will happen only after the GitHub version has a complete, visible public foundation ready to receive the domain.

GitHub is the first public build layer.

Cloudflare is a later DNS, performance, and security layer.

## Governing Sentence

**Classify the evidence. Do not perform the verdict.**
