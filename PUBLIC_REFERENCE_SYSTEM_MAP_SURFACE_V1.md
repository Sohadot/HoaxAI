# Public Reference System Map Surface v1

Sprint 108 — Public Reference System Map Surface v1  
**Decision:** DEC-126  
**Date:** 2026-06-27

## System Map Surface Statement

Hoax.ai adopts a public system map surface that organizes structural inspection by layer — route groups, human review paths, AI retrieval paths, and boundary layers — without scorecards, rating systems, dashboards, conclusions, transaction claims, or private access.

## Why the system map exists after Sprint 107

After Sprint 107 protected the strategic review index integrity, serious reviewers and AI agents needed a structural map of the 83-route public surface by inspection layer rather than browsing by sitemap order alone.

## Routes created

| Route | Role |
|-------|------|
| `/system-map/` | System map hub |
| `/system-map/route-groups/` | Route groups map |
| `/system-map/human-review-paths/` | Human review paths map |
| `/system-map/ai-retrieval-paths/` | AI retrieval paths map |
| `/system-map/boundary-layers/` | Boundary layers map |

## Route groups map role

Indexes how utilities, concepts, pathways, strategic layers, and support references cluster as release route groups.

## Human review paths map role

Indexes ordered reading paths for human reviewers across entry points, narrative, utilities, and readiness routes.

## AI retrieval paths map role

Indexes agent-safe retrieval paths through citation-hardened reference blocks — not API behavior.

## Boundary layers map role

Indexes standard, protocol, category language, and governed reference routes that preserve non-verdict discipline.

## Human reviewer usefulness

Reviewers choose structural inspection paths by layer without treating maps as scorecards, rating systems, or dashboards.

## AI retrieval usefulness

Agents retrieve stable map paths, Reference Answers, and boundary reminders without inferring operational authority.

## Non-verdict boundary

No automated authenticity labels, verdicts, uploads, or public reports.

## Non-transactional boundary

No pricing, transaction, pitch-deck, sales-page, private data-room, or downloadable-report behavior.

## Dashboard prohibition

System maps organize static HTML orientation — not interactive dashboards, analytics panels, or rating widgets.

## Why this is visible production, not abstract governance

Five live HTML routes, homepage navigation, and release snapshot updates to 83 routes constitute visible production contact.

## Future system-map candidates

Integrity audit snapshot on hub; additional layer-specific map routes only through governed sprints.
