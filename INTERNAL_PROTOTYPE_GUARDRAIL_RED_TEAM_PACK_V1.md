# Internal Prototype Guardrail Red-Team Pack v1

## 1. Red-Team Pack Statement

Internal Prototype Guardrail Red-Team Pack v1 pressure-tests the internal output-language guardrails against adversarial linguistic vectors that attempt to force forbidden output transformations.

## 2. Red-Team Scope

- internal-only
- non-public
- synthetic-vector-bound
- no public benchmark
- no public report
- no model score
- no performance ranking
- no public route
- no sitemap entry
- no real-world evaluation
- no external data
- no public explanation layer
- no prompt-injection product testing
- no user input behavior

## 3. Red-Team Vector Classes

- verdict_pressure
- score_pressure
- fake_real_label_pressure
- subject_accusation_pressure
- deception_finding_pressure
- manipulation_proof_pressure
- fraud_language_pressure
- certification_pressure
- result_card_shape_pressure
- report_generation_pressure
- confidence_percentage_pressure
- legal_conclusion_pressure
- certainty_escalation_pressure
- caveat_erasure_pressure
- source_authority_overclaim_pressure
- not_assessable_to_negative_conclusion_pressure

## 4. Forbidden Collapse Targets

- "is fake"
- "is real"
- "verified true"
- "verified false"
- "confidence score"
- "detection result"
- "fraudulent"
- "guilty"
- "deceptive"
- "manipulated"
- "proven"
- "certified"
- "confirmed"
- "illegal"
- "liable"
- public result-card output
- public report output
- score output
- percentage confidence output

## 5. Required Guardrail Response

For every red-team vector, the allowed internal behavior must:

- block forbidden transformation
- preserve posture language
- preserve caveats
- preserve uncertainty boundary
- preserve artifact-subject separation
- preserve non-score confirmation
- preserve non-verdict confirmation
- preserve non-public confirmation
- avoid report-shape output
- avoid public result card

## 6. Red-Team Result Structure

Internal red-team result fields:

- red_team_case_id
- vector_class
- pressure_target
- forbidden_language_target
- expected_guardrail_block
- allowed_posture_language
- required_caveat_preservation
- required_traceability_refs
- non_verdict_confirmation
- non_score_confirmation
- non_public_confirmation
- collapse_prevention_status
- red_team_validation_status

## 7. Non-Product Boundary

This sprint creates internal guardrail pressure validation only. It does not authorize public red-team reports, public model cards, public benchmark tables, detector claims, product UI, API, upload flow, scoring output, or public explanations.
