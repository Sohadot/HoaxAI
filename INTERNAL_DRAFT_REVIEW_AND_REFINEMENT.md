# Hoax.ai Internal Draft Review and Refinement

**Version:** v1.0.0  
**Status:** governed_internal_draft_review_refinement  
**Maturity:** internal_review_only_no_routes_no_sitemap_no_publication  
**Decision:** DEC-040

## A. Purpose

This document governs the review and refinement of non-site-public internal draft files before any public route, sitemap, public metadata, public navigation, or release pathway may be considered.

## B. Non-Purpose

This review and refinement does **not**:

- create public pages;
- create new internal draft files;
- create route files;
- activate routes;
- expand sitemap.xml;
- create public metadata;
- create public navigation links;
- create SEO expansion;
- create public classifier functionality;
- create a public tool;
- create upload functionality;
- create scoring;
- create forms;
- create analytics;
- deploy anything;
- enable DNS or Cloudflare;
- authorize publication.

## C. Governing Principle

**Review improves the draft. It does not move the draft toward publication.**

**Refinement may strengthen internal substance, but it must not create public readiness claims.**

## D. Review Definition

Internal draft review is a non-public governance review of existing internal draft files against required section contracts, reference substance, claim/source boundaries, semantic SEO discipline, prohibited misreadings, structured data boundaries, interface boundaries, security/privacy boundaries, and publication-blocking status.

## E. Refinement Definition

Refinement is controlled improvement of existing internal draft prose to strengthen precision, section completeness, boundary discipline, claim/source clarity, internal reference value, and non-authorization language without creating public readiness, route eligibility, sitemap eligibility, public metadata eligibility, or publication eligibility.

## F. Review Is Not

Review is not:

- publication approval;
- route approval;
- sitemap approval;
- public SEO approval;
- public metadata creation;
- source verification;
- truth certification;
- external factual validation;
- public release readiness;
- deployment readiness.

## G. Review Scope

| Draft File | Draft ID | Candidate |
|------------|----------|-----------|
| `_internal_drafts/reference/evidence-posture.md` | DRAFT-0001 | REF-CAND-0001 |
| `_internal_drafts/reference/artifact-subject-separation.md` | DRAFT-0002 | REF-CAND-0002 |

**Excluded:** Output Boundary draft, Claim and Source Traceability draft, Synthetic Fragility draft, any new draft file, any public page.

1. Draft Status Integrity  
2. Required Section Completeness  
3. Reference Substance  
4. Definition and Scope Clarity  
5. Governance Boundary Clarity  
6. Claim Scope Discipline  
7. Source Scope Discipline  
8. Semantic SEO Boundary Discipline  
9. Prohibited Misreading Protection  
10. Internal Link Plan Discipline  
11. Structured Data Boundary  
12. Interface Boundary  
13. Security/Privacy Boundary  
14. Non-Authorization Strength  
15. Forbidden Language Safety  
16. Public Surface Safety  
17. Word Count and Substance Control  

## I. Review Outcomes

Allowed outcomes: `review_passed_internal`, `review_passed_with_refinement`, `needs_minor_refinement`, `needs_major_refinement`, `blocked_for_boundary_issue`, `blocked_for_claim_source_issue`, `blocked_for_public_implication`.

No outcome may authorize route creation, sitemap expansion, public metadata, publication, or deployment.

## J. Refinement Rules

Refinement may clarify definitions, strengthen governance boundary, improve section flow, remove generic phrasing, remove public-tool implication, remove detector/scoring/upload/fake-real framing, strengthen non-authorization statement, tighten semantic SEO role, improve internal link plan language, improve claim/source scope clarity, and improve structured data boundary language.

Refinement must not add external factual claims, public titles/meta/canonicals/schema, public route language, sitemap language, deployment language, service/product/software/tool claims, or examples involving real people, companies, institutions, political events, current events, or accusations.

## K. Post-Sprint Status

After Sprint 22, drafts may have review records and refinement logs, may be internally improved, and remain `not_route_created`, `not_sitemap_eligible`, `publication_blocked`, and not public pages. Future public route readiness remains blocked until a later explicit sprint.
