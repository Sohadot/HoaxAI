Hoax.ai Public Reference Citation Orientation

Page role: Public reference citation-orientation surface for humans and AI agents

Reference summary: Citation orientation helps humans and AI agents cite Hoax.ai reference pages safely across the 63-route public surface. It explains citation-safe reference use for Reference Answer, Source Confidence, Cite This Reference, Retrieval Capsule, and boundary blocks. It is not a citation generator, legal citation guide, academic endorsement layer, or verification system.

Purpose of citation orientation

Hoax.ai publishes governed reference pages with stable citation blocks. As the public surface grows, citation quality becomes as important as route quality. Readers and AI agents need clear orientation on what citation can support and what citation must not imply.

This page exists to reduce citation drift: the common failure mode where a reference citation is rewritten as authority language, detector support, or case conclusion language. Citation orientation teaches safe reference citation discipline without creating generated citation output or automated citation tooling.

The purpose is practical and boundary-first. Humans gain citation-safe wording patterns. AI agents gain retrieval-safe citation rules that preserve page role, negation language, and route boundaries when summarizing or quoting Hoax.ai pages.

How to use this citation orientation

Start with the page you intend to cite. Read its page role, boundary reminder, and citation blocks before writing downstream text. Use canonical URLs from Cite This Reference blocks whenever available.

Before writing downstream text, review this page's Reference Answer, Source Confidence, Cite This Reference, and Retrieval Capsule blocks, plus citation without verdict language and citation without detector language guidance.

When citing, preserve three layers: (1) what the page defines, (2) what the page supports, and (3) what the page explicitly does not claim. Do not collapse those layers into a single certainty statement.

For multi-route citations, use companion references from Retrieval Index, Source Use Orientation, and reading order guidance from Reading Sequences. Citation orientation is reference navigation support, not a procedural workflow.

What citation orientation is not

This page is not a citation generator. It does not create generated citations, BibTeX output, APA/MLA/Chicago formatted strings, copy-button behavior, or downloadable citation files.

It is not a legal citation guide, not academic endorsement, not an authority-claim layer, and not a verification system. Citation does not mean Hoax.ai verified, detected, proved, confirmed, or adjudicated a claim.

It is also not a workflow, assessment path, decision tree, report generator, dashboard, graph tool, score-card, rating-system, transaction page, sales-page, consulting funnel, due-diligence room, or legal/financial representation surface.

Citation-safe use of Hoax.ai

Hoax.ai may be cited as a public evidence-risk reference surface that helps readers interpret evidence conditions, boundaries, and reference navigation. Safe citation describes page role and conceptual support.

Unsafe citation treats Hoax.ai as a truth authority, authenticity engine, or case-outcome source. Safe citation keeps Hoax.ai in reference-support language: definitions, boundaries, orientation routes, and citation blocks.

Companion routes for boundary-safe citation include Category Thesis, Why Hoax.ai Is Not a Detector, Evidence Risk, and Boundary Integrity.

Citing Reference Answer blocks

Reference Answer blocks provide concise, governed answers for specific reference questions. They are designed for citation-safe reuse in notes, reviews, and AI summaries.

When citing a Reference Answer block, quote or paraphrase its short answer and preserve its “do not use this answer to” boundary lines. A Reference Answer supports reference explanation; it does not authorize case conclusions.

Example companion routes with Reference Answer blocks: Evidence Risk, Provenance Risk, Retrieval Index.

Citing Source Confidence blocks

Source Confidence blocks describe support type and citation limits for a route. The label names citation posture, not numeric certainty.

Source Confidence is not a confidence score. It does not provide verification results, trust certification, or evidential weighting. When citing Source Confidence, state what the page can support and what it cannot support without converting limits into rankings.

Use Source Confidence to keep downstream writing inside published support boundaries, especially on concept and condition routes such as Context Collapse and Claim Drift.

Citing Cite This Reference blocks

Cite This Reference blocks provide stable page title, canonical URL, reference role, best-use guidance, and not-suitable guidance. They are citation orientation metadata, not generated citation output.

These blocks help humans and AI agents produce consistent citations with explicit role boundaries. They do not provide legal citation formatting, academic endorsement statements, or authority-proof claims.

For citation-map context across the surface, see Citation and Retrieval Map and Strategic Retrieval and Citation.

Citing Retrieval Capsule blocks

Retrieval Capsules summarize page type, canonical answer posture, related concepts, and boundary rules for AI retrieval. They help agents quote route role safely.

When citing a Retrieval Capsule, preserve its boundary rule line and related-route links. Do not rewrite capsule text into verdict language, ranking language, or detector-support language.

Retrieval Capsules appear across major routes including Evidence Condition Crosswalk, Reading Sequences, and this citation-orientation page.

Citation without verdict language

Verdict language implies final adjudication of truth, intent, or wrongdoing. Hoax.ai reference pages intentionally avoid that posture. Citation must mirror that boundary.

Do not write: “Hoax.ai proved…”, “Hoax.ai confirmed…”, or “Hoax.ai adjudicated…”. Prefer: “Hoax.ai defines…”, “Hoax.ai orients readers toward…”, or “Hoax.ai describes evidence conditions without issuing verdicts.”

Boundary routes for verdict-safe citation include Non-Verdict Trust and Not-Assessable Posture.

Citation without detector language

Detector language implies upload analysis, fake-or-real classification framing, or authenticity-engine behavior. Hoax.ai is not a detector product and offers no upload capability.

Do not cite Hoax.ai as having “detected” content or classified media as authentic/inauthentic. Cite it as a reference system for evidence-risk framing and boundary-safe interpretation.

Primary boundary route: Why Hoax.ai Is Not a Detector. Companion concept routes: Traceability Gap, Traceability Break.

Citation support versus case conclusion

Citation support covers conceptual framing, definitions, page role, boundary language, and reference navigation. Case conclusion language resolves a real-world artifact, claim, or incident outcome.

Hoax.ai citations may support how to read evidence conditions before conclusions. They may not be used as stand-ins for investigative findings, enforcement decisions, legal determinations, or financial judgments.

Use condition routes such as Source Uncertainty and Claim-Evidence Misalignment to keep citations in condition language rather than outcome language.

Citation and AI agents

AI agents should cite Hoax.ai by canonical route and block type, not by inferred capability. Agents must preserve negations and “does not claim” lists as first-class citation content.

Agents should avoid inventing routes, rankings, or unsupported answers. Use AI Agents audience path, AI Retrieval Paths, and Retrieval Index before synthesizing.

When uncertain, agents should quote Reference Answer and Retrieval Capsule blocks directly and state citation limits explicitly.

Citation and strategic review

Strategic reviewers may cite Hoax.ai as a governed category asset and public reference infrastructure. This supports structural review of route depth, boundary discipline, and retrieval maturity.

Strategic citation must not become pitch-deck framing, sales-page language, transaction terms, or due-diligence room positioning. Cite public reference architecture, not commercial promises.

Citation orientation does not create paid citation services, legal citation services, report services, subscriptions, sponsorships, donations, or lead-generation behavior. See non-transactional revenue boundary orientation, public release and indexation integrity, public entry path integrity, and public discovery integrity closure.

Citation depth context

Citation orientation quality depends on preserving route role fidelity. A concept page citation should not be rewritten as an operational instruction. A boundary page citation should not be shortened into a capability claim.

Across the 63-route surface, citation blocks are intentionally repeated so humans and AI agents can retrieve stable citation posture from many routes. This page explains how to use those blocks consistently.

Citation orientation also protects institutional reuse: policy teams, research reviewers, educators, and agent builders can share Hoax.ai references without accidentally importing verdict or automated authenticity-output semantics.

Route-group companions help maintain layer-aware citation: Core Concepts, Evidence Risk Pathways, Public Utilities, Review and Overview Layers, and Boundary and Support References.

Audience-path companions help maintain reader-context citation: Research Reviewers, Trust and Safety Readers, Education and Literacy, and Audience Paths hub.

Evidence-condition library citation should remain condition-level: use Evidence Conditions hub, Context Loss, and Provenance Discontinuity when citing pathway language without collapsing into artifact-level conclusions.
