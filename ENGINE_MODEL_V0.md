# Evidence Posture Engine Model v0

## 1. Model Statement

**Evidence Posture Engine Model v0 maps evidence conditions, protocol steps, posture states, boundary checks, and permitted output language without executing review, accepting inputs, generating results, assigning scores, or producing verdicts.**

This model is a governed internal representation only. It does not run, serve, classify, score, or emit outputs to any public surface.

## 2. Model Status

Evidence Posture Engine Model v0 is:

- **internal model only** — repository documentation and JSON, not a deployed system
- **non-operational** — no execution path, no runtime, no automation
- **no public engine** — Hoax.ai has no public engine and none is authorized by this model
- **no input system** — no artifact intake, no submission pipeline
- **no output generator** — no automated language emission
- **no classifier** — no categorical verdict labels beyond governed posture vocabulary
- **no score** — no numeric ranking or confidence percentage
- **no API** — no endpoints, no integration surface
- **no UI** — no user-facing engine interface

No public engine currently exists. No public engine is authorized by this sprint.

## 3. Source Authority

Evidence Posture Engine Model v0 is derived from:

- **Evidence Posture Standard v1** — EPS-001 through EPS-014 governing principles and posture vocabulary
- **Evidence Posture Protocol v1 Draft** — EP-P01 through EP-P17 ordered review steps
- **Engine Boundary Charter** — engine drift boundaries and non-operational constraints
- **Output Boundary** — permitted and prohibited governed statement scope
- **Evidence Limitation** — limitation-as-credibility, not falsehood substitution
- **Attribution Boundary** — artifact-only scope without subject transfer
- **Interpretation Risk** — restraint without accusation
- **Artifact–Subject Separation** — artifact condition is not subject guilt
- **Claim–Source Traceability** — claim strength relative to cited basis

The model subordinates any future engine architecture to these authorities. It may not exceed the standard, protocol, or charter.

## 7. Evidence Condition Dimensions

The model tracks twelve evidence condition dimensions:

- artifact_condition
- claim_condition
- source_confidence
- provenance_status
- context_status
- traceability_status
- chain_status
- drift_status
- limitation_status
- interpretation_risk_status
- attribution_boundary_status
- output_boundary_status

Dimensions are descriptive states for mapping—not scores, ranks, or verdict labels.

## 8. Posture States

Evidence Posture Engine Model v0 uses exactly five posture states from the standard:

- **Supported**
- **Qualified**
- **Limited**
- **Not Assessable**
- **Out of Scope**

**No numeric score. No percentage. No fake/real label.**

Posture assignment is categorical discipline aligned with EP-P16 and EPS-005. The model does not implement a verdict system.

## 9. Boundary Checks

| Check ID | Purpose |
|----------|---------|
| artifact_subject_separation_check | Block artifact-to-subject guilt transfer |
| source_confidence_not_certification_check | Source relationship without truth/fraud certification |
| provenance_gap_not_manipulation_check | Gap narrows language; does not prove manipulation |
| context_collapse_not_motive_check | Context loss without motive or misconduct proof |
| claim_drift_not_deception_check | Drift naming without deception default |
| evidence_limitation_not_falsehood_check | Limitation without falsehood substitution |
| interpretation_risk_not_verdict_check | Restraint without blame or accusation |
| attribution_boundary_no_subject_transfer_check | Artifact-only scope preserved |
| output_boundary_no_forbidden_language_check | Refuse language the standard prohibits |

Boundary checks are model gates for future governed review—not automated enforcement.

## 10. Permitted Output Types

Only these output types may be modeled:

- posture_state
- evidence_condition_summary
- source_caveat
- provenance_caveat
- context_caveat
- traceability_caveat
- limitation_caveat
- interpretation_risk_caveat
- attribution_boundary_caveat
- bounded_language_template

Permitted types align with EP-P17 bounded output formation. They describe language categories, not generated text.

## 11. Prohibited Output Types

The model must prohibit:

- fake/real verdict
- truth/falsity verdict
- subject guilt
- deception intent
- manipulation proof
- fraud accusation
- responsibility assignment
- legal conclusion
- moderation action
- numeric score
- confidence percentage
- upload classification result
- automated enforcement recommendation

**No score, no fake/real label, and no verdict system.**

## 12. Model Failure Modes

| Failure mode | Description |
|--------------|-------------|
| detector drift | Model treated as fake/real detector |
| scoring drift | Numeric or percentage confidence introduced |
| upload implication | Model implies upload or submission workflow |
| fake/real leakage | Authenticity certification language |
| subject accusation leakage | Artifact signals transferred to subjects |
| evidence limitation converted into falsehood | Gap treated as proof of false content |
| claim drift converted into deception | Structural drift treated as deceptive intent |
| interpretation risk converted into verdict | Restraint language becomes accusation |
| source weakness converted into fraud | Source condition treated as fraud label |
| provenance gap converted into manipulation | Missing chain treated as concealment proof |

## 13. Future Model Gate

A future internal prototype may be considered only after Engine Model v0 validation passes, Output Language Guardrail Model v1 validation passes (DEC-087), Internal Non-Public Engine Prototype Charter validation passes (DEC-088), and a separate sprint explicitly authorizes controlled non-public prototyping.

## Output Language Guardrail Dependency

Evidence Posture Engine Model v0 maps evidence conditions and protocol structure. Output Language Guardrail Model v1 (Sprint 69, DEC-087) maps **allowed and prohibited output language**, linguistic primitives, posture-state grammar, and boundary transformation rules. Any future engine-facing output must obey both the model and the guardrail. See `OUTPUT_LANGUAGE_GUARDRAIL_MODEL_V1.md` and `data/output-language-guardrail-model-v1.json`.

## Internal Non-Public Engine Prototype Charter Dependency

Internal Non-Public Engine Prototype Charter (Sprint 70, DEC-088) defines admissibility conditions, fixture policy, environment boundaries, execution limits, and future authorization gates for any later controlled internal prototype. Any future prototype must obey this charter in addition to Engine Model v0 and Output Language Guardrail Model v1. See `INTERNAL_NON_PUBLIC_ENGINE_PROTOTYPE_CHARTER.md` and `data/internal-non-public-engine-prototype-charter-v1.json`.
