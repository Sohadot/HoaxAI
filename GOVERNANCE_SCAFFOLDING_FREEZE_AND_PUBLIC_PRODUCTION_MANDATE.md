# Governance Scaffolding Freeze and Public Reference Production Mandate

## A. Diagnosis

Hoax.ai's governance foundation is strong. The decision log, claim policy, evidence posture standard, classification protocol, publisher control plane, reference candidate pipeline, and public route governance chain represent genuine category infrastructure. These layers protect the asset from legal, reputational, and conceptual weakness.

However, recent sprints — from Sprint 44 through Sprint 51 — have created successive governance layers whose primary output is a future governance layer. Public Route Eligibility Governance produced a validation. That validation produced Candidate Assessment Governance. That produced Candidate Assessment Governance Validation. That produced Registry Governance. That produced Registry Governance Validation. That produced Registration Governance. That produced Registration Governance Validation. And that was pointing toward Registration Authorization Governance.

None of these layers produced a public reference page.

The project must stop producing governance layers whose main output is another future governance layer. Governance has begun to replace production rather than protect it.

## B. Governing Correction

> **Governance must protect production, not replace production.**

> **No further meta-governance layer may be added until Hoax.ai has produced at least 10 additional public reference pages.**

These are the two governing sentences of this mandate. They override the direction inherited from Sprint 51.

## C. Frozen Work Types

The following work types are frozen effective Sprint 52:

- Public Route Candidate Registration Authorization Governance
- Candidate Registration Authorization Validation
- Candidate Registry Expansion Governance
- Route Eligibility Abstraction Layers
- Governance Validation of Governance Validation
- Future Public-Readiness Abstraction Layers
- Any sprint whose main output is only another permission framework for a later framework

Frozen means: no new sprint may be started whose primary deliverable is one of these types. The existing governance chain (DEC-044 through DEC-069) is preserved but not extended further.

## D. Allowed Immediate Work

The following work types are authorized immediately:

- Public reference page creation
- Route registry entries for real public reference pages
- Sitemap expansion for those real pages
- Internal links between real pages
- Metadata for real pages
- Source/claim mapping where needed for new public pages
- Validators that protect real page quality
- Audits proving real page quality

All other governance scaffolding expansion is blocked until the production threshold is met.

## E. Production Threshold

```
minimum_public_reference_pages_before_new_meta_governance: 10
```

Hoax.ai currently has the following public reference pages:
1. /reference/evidence-posture/
2. /reference/artifact-subject-separation/
3. /language/

These count toward the total public surface but not toward the 10 additional reference pages required before any new meta-governance layer may be proposed.

The 10 additional pages must be distinct reference pages with governed purpose, route registry alignment, sitemap inclusion, internal linking, claim/source discipline where needed, metadata, accessibility, and validation.

## F. Quality Boundary

Every new public reference page must:

1. Have a clear public purpose — it must teach a real Hoax.ai concept
2. Teach a real Hoax.ai concept — evidence posture, artifact risk, source discipline, provenance, or related taxonomy
3. Preserve evidence posture, not truth verdict — no final judgments about people, events, or claims
4. Preserve artifact-subject separation — classify artifacts, not persons or institutions
5. Avoid detector/scanner/fake-real/scoring/upload implications — no tool, product, or service language
6. Include sufficient reference substance — no thin content, no placeholder pages
7. Contain exactly one H1 — matching the page topic
8. Include metadata and canonical — title, description, canonical URL
9. Be registered in route registry — before any public surface exposure
10. Be included in sitemap only after route validation — no premature sitemap entries
11. Be internally linked from relevant existing pages — strengthen link graph
12. Pass validate_all.py — full validator suite must pass before release

## G. Next Production Candidates

The following reference pages are the recommended first production batch:

1. `/reference/source-confidence/` — What it means for a source to carry confidence weight in evidence posture analysis
2. `/reference/provenance-gap/` — What happens when origin, chain of custody, or creation context cannot be established
3. `/reference/not-assessable/` — Why some artifacts cannot be assessed and what this state means for evidence posture
4. `/reference/output-boundary/` — The scope and limits of evidence posture outputs as defined in Hoax.ai's output boundary schema
5. `/reference/synthetic-fragility/` — The structural characteristics that make synthetic media artifacts detectable through provenance and context rather than pixel analysis
6. `/reference/evidence-chain/` — How evidence posture reasoning connects artifact, context, source, and provenance
7. `/reference/context-collapse/` — What happens when an artifact is separated from its original context and how this affects evidence posture
8. `/reference/claim-source-traceability/` — Why every material claim in a reference context must be traceable to a source
9. `/reference/evidence-limitation/` — The bounded nature of evidence posture assessment and why limitations are features, not weaknesses
10. `/reference/classification-boundary/` — What Hoax.ai classifies and what it does not classify

These are candidates, not approved routes. Each must pass route registry, claim/source review, content quality review, and validate_all.py before public release.

## H. Non-Authorization

This production reset does not authorize:

- Engine development or deployment
- Classifier development or deployment
- Upload workflow of any kind
- Scoring or confidence-score interfaces
- API development or deployment
- Analytics integration
- DNS or Cloudflare changes
- Custom domain launch
- Monetization
- Public tool behavior of any kind
- Workbench launch or public workbench exposure
- Any sprint whose primary output is a new permission layer for a later phase

Hoax.ai's next authorized sprint is public reference page production. That sprint creates real pages. It does not create permission frameworks for future page creation.
