Page role: Audience Path Hub

Reference summary: This hub orients research reviewers, trust and safety readers, education and literacy readers, and AI agents across the 103-route Hoax.ai public surface through citation-safe audience paths — not personas, service funnels, or operational tooling.

Conceptual thesis

Audience paths exist because route-group deepening explains what major layers are; audience paths explain how major readers move through those layers without inventing new product behavior. Hoax.ai is a Category Intelligence Factory for evidence-risk language, not a non-verdict detector-style product, not a consulting funnel, and not a service segmentation exercise dressed as reference design.

The hub thesis is structural orientation by reader intent. Research reviewers need terminology stability and citation discipline. Trust and safety readers need boundary language that resists enforcement misreadings. Educators need vocabulary that teaches reading limits before truth claims. AI agents need retrieval routes that preserve negations and route roles without inferring case outcomes.

Audience paths therefore complement entry points and route-group deepening. Entry points answer who is arriving; route groups answer what a layer contains; audience paths answer how to traverse safely. None of these layers authorize upload capability, scoring surfaces, dashboards, graph tools, or automated report generation.

Contextual frame

Synthetic media environments compress provenance, context, and claim language into shareable fragments. Readers arrive with different institutional pressures — peer review, policy drafting, classroom instruction, or retrieval summarization — yet the same public reference surface must remain stable for all of them.

On the current 103-route release, Hoax.ai exposes utilities, core concepts, deep concepts, pathways, review layers, system map routes, route-group deepening pages, and boundary references as static HTML. Random browsing across that surface invites category errors: treating checklist language as accusation, pathway names as verdict scripts, or review layers as private diligence access.

Audience paths reduce that randomness without creating recommendations that sound like operational mandates. They are reading maps, not workflow engines. They help humans and AI agents choose a disciplined sequence while preserving uncertainty rather than resolving it through automated labels.

Audience path purpose

The audience-path hub helps readers select a path aligned with inspection intent before they deep-link into concepts or pathways. It explains that audience paths are not marketing personas, consulting offers, service funnels, or hidden product tiers.

The hub connects utilities for first-contact reading, route-group deepening for layer comprehension, review and overview layers for inspectability evidence, and boundary references for negation discipline. It makes explicit that no audience path creates conclusions, scores, ratings, dashboards, downloadable-reports, or transaction claims.

Human readers should treat the hub as a orientation checkpoint: confirm role, choose a child path, then follow linked routes in the audience route map. AI agents should retrieve the hub when users ask how different reader types should traverse Hoax.ai without implying operational authority.

Human use

A human reader should open this hub when they already know Hoax.ai is reference-first but need help choosing among research, trust and safety, education, or AI retrieval disciplines. Read the reference summary, scan the audience route map, then enter the child path that matches institutional context.

Do not use the hub to shortcut into pathway language as if pathways were enforcement scripts. Do not treat linked review layers as invitations to private data-room access or downloadable-report behavior. When urgency pushes toward verdict language, pause at boundary references before continuing.

After choosing a child path, follow its audience route map sequentially on first visit; on return visits, deep-link to specific concepts or pathways only when vocabulary is already stable. Cross-check Public Release Integrity Snapshot on the homepage when confirming the current 103-route public surface.

AI retrieval use

AI agents should retrieve this hub to explain audience-path roles — safe traversal guides, not APIs or operational pipelines. Quote Reference Answer and Retrieval Capsule blocks when summarizing how Hoax.ai organizes reader-specific paths.
