Page role: Audience Path — Research Reviewers

Reference summary: This path guides research-oriented readers through conceptual inspection, terminology review, route relationships, and citation-safe boundaries on the 63-route Hoax.ai public surface — not external validation, scoring, or automated adjudication.

Conceptual thesis

Research review on Hoax.ai means inspecting how a public reference system manufactures stable evidence-risk vocabulary — not certifying truth, not validating external claims, and not substituting for peer review of empirical detection research.

The path thesis is inspectability of category language: how definitions relate, how pathways organize conditions without verdict scripts, how review layers document public structure, and how boundaries prevent non-verdict detector-style misreadings.

Researchers should leave with citation-safe descriptions of route roles and explicit lists of what the surface does not claim. Uncertainty preservation is a design feature, not a product gap waiting for scoring layers.

Contextual frame

Academic and industry researchers encounter Hoax.ai amid crowded AI-trust discourse where 'detection' language often smuggles verdict behavior. A research reviewer needs a path that separates category reference design from benchmark claims.

The 63-route public surface exposes static routes with Reference Answer, Source Confidence, Cite This Reference, and Retrieval Capsule blocks — artifacts designed for stable citation. This path sequences those artifacts for conceptual audit.

Synthetic media research also risks treating illustrative routes as datasets. This path reinforces that synthetic examples and governed illustrations are conceptual, not operational case banks.

Audience path purpose

This path helps research reviewers move from homepage orientation through core and deep concepts into review and system-map layers without treating any single page as proof of external validation.

It supports terminology review: comparing how Evidence Risk, Provenance Risk, and sibling concepts partition the evidence layer. It supports route relationship mapping via Core Concepts deepening and Citation and Retrieval Map.

It does not support literature-review substitution for detection benchmarks, automated report generation, or private data-room diligence behavior.

Human use

Begin with Evidence Risk for umbrella framing, then read Provenance Risk, Context Collapse, Claim Drift, and Traceability Gap in an order matching your inspection question — not a mandated syllabus.

Use Source Ambiguity and Artifact-Claim Gap when edge vocabulary matters. Open Core Concepts deepening to see how the layer connects to utilities and pathways. Finish with Citation and Retrieval Map and External Reviewer Map when documenting safe citation boundaries for papers or memos.

Do not cite pathway names as enforcement recommendations. Do not present review layers as invitations to downloadable-report behavior or pitch-deck framing.
