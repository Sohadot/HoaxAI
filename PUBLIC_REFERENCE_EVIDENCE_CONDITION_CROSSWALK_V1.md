# Public Reference Evidence Condition Crosswalk v1

**Decision:** DEC-133
**Sprint:** Sprint 115
**Status:** Adopted — visible public production

## Evidence Condition Crosswalk Statement

Public Reference Evidence Condition Crosswalk v1 adds one public reference route, `/evidence-conditions/crosswalk/`, that relates the five evidence conditions in the Evidence Condition Library to core concepts, evidence-risk pathways, route groups, audience paths, boundary language, and AI retrieval use.

Governing sentence: the crosswalk relates evidence conditions across the public reference system; it does not rank, score, verify, detect, or adjudicate them.

## Why the crosswalk exists after the Evidence Condition Library

Sprint 114 introduced five governed evidence conditions, each defined on its own citation-safe page. Readers who meet a condition in isolation still need placement knowledge: which concepts define its vocabulary, which pathways organize its reading routes, which route groups deepen it, which audience paths route its likely readers, and which boundary language keeps it citation-safe. The crosswalk answers those placement questions on one page so relations are stated by the surface rather than invented by readers or retrieval systems.

## Route created

- `/evidence-conditions/crosswalk/` — ROUTE-0100, PUB-FILE-0100

The public surface increases from 99 to 100 routes. The sitemap increases from 99 to 100 URLs.

## Crosswalk role

The crosswalk is a relation surface, not an operational surface. It relates the five conditions — Source Uncertainty, Provenance Discontinuity, Context Loss, Claim–Evidence Misalignment, and Traceability Break — to the rest of the reference system. The hub `/evidence-conditions/` is treated as the library orientation page, not as a condition itself.

## Relation language

The crosswalk uses only non-ranking relation labels:

- directly relevant
- contextually relevant
- boundary-sensitive
- retrieval-supporting
- confusion-preventing
- adjacent reference

No relation implies severity, weighting, ordering, likelihood, or outcome.

## Human reader usefulness

Human readers use the crosswalk to choose companion reading for a named condition: the concept page for definition, the pathway for reading order, the route group for shelf placement, the audience path for reader-specific orientation, and the boundary pages for citation safety.

## AI retrieval usefulness

AI agents cite the crosswalk when users ask how conditions relate to each other or to the wider surface. The crosswalk prevents invented orderings between conditions and routes agents toward the stable Reference Answer and Retrieval Capsule blocks on each condition page.

## Non-verdict boundary

The crosswalk creates no verdicts, scores, ratings, severity orderings, risk matrices, detector outputs, dashboards, or graph tools. Naming a relation between a condition and another reference layer implies nothing about the truth status of any artifact or claim.

## Non-transactional boundary

The crosswalk is not a transaction page, pricing statement, acquisition term document, representative mandate, legal representation, financial representation, private due-diligence room, downloadable-report, pitch-deck, sales-page, consulting offer, or service funnel.

## Why this is visible production, not abstract governance

The crosswalk is a public, indexable, human-readable route linked from the homepage, the evidence-condition hub, the system map, route-group pages, and audience-path pages. DEC-133 governs this visible page; it is not preparatory governance for a future layer.

## Future crosswalk candidates

Future crosswalk expansion — such as relating conditions to utilities or review layers in more depth — requires governance gates and expansion approval before any new route is created.
