# Hoax.ai Content Quality and Reference Substance Standard

**Version:** v1.0.0  
**Status:** governed_internal_content_quality_standard  
**Maturity:** pre_reference_publication_standard  
**Decision:** DEC-032

## A. Purpose

This standard defines minimum content quality and reference substance for future Hoax.ai reference pages.

## B. Non-Purpose

This standard does **not**:

- create public pages;
- create draft pages;
- create routes;
- create SEO expansion;
- create sitemap expansion;
- create publisher output;
- create deployment readiness;
- create DNS or Cloudflare work;
- create public classifier functionality;
- create a public tool;
- create upload functionality;
- create forms;
- create analytics;
- create monetization;
- create external factual claims.

## C. Governing Principle

**A page is not reference-grade because it is long. It is reference-grade because it carries governed substance.**

**No public reference page may exist unless its substance can be inspected, mapped, bounded, and validated.**

**SEO is allowed only when it clarifies governed reference value, not when it substitutes for it.**

## D. Definition of Reference Substance

Reference substance is the governed combination of purpose, definition, scope, boundary, system relationship, claim traceability, source scope, semantic SEO discipline, internal link logic, prohibited misreadings, review status, and technical quality that makes a page useful as part of Hoax.ai's reference layer.

## E. Substance Is Not

Substance is not:

- word count alone;
- keyword density;
- decorative prose;
- generic AI commentary;
- topical news reaction;
- repeated slogans;
- thin glossary text;
- search-volume capture;
- unsourced external claims;
- invented authority;
- fake/real examples;
- sensational fraud framing;
- tool-promise language.

## F. Minimum Substance Gates

Every future reference page must pass:

1. **Purpose Gate** — why the page exists in Hoax.ai's category system.
2. **Definition Gate** — defines the governed concept, term, state, dimension, boundary, or framework.
3. **Scope Gate** — what it covers and does not cover.
4. **Governance Boundary Gate** — what it must not be read to claim.
5. **System Relationship Gate** — connects to taxonomy, standard, protocol, output boundary, governance boundary, claim/source traceability, interface governance, security/privacy boundary, reference expansion gate, or publisher control plane.
6. **Claim Traceability Gate** — major claims map to evidence ledger and claim-source map.
7. **Source Scope Gate** — external facts require registered source support.
8. **Semantic SEO Discipline Gate** — metadata clarifies the concept without unsupported claims or tool implications.
9. **Prohibited Misreading Gate** — prevents truth-machine, detector, upload, scoring, and accusation misreadings.
10. **Internal Link Gate** — links only to registered eligible routes.
11. **Technical Quality Gate** — passes technical quality validation.
12. **Interface Boundary Gate** — no scanner/detector/scoring visual logic.
13. **Publisher Boundary Gate** — not produced by publisher automation without dry-run harness and approval.
14. **Review Status Gate** — includes last reviewed or review status metadata.

## G. Required Page Section Classes

**Required:** Page Thesis; Definition and Scope; Governance Boundary; Relationship to Hoax.ai System; Semantic SEO Role; Prohibited Misreadings; Claim and Source Traceability; Internal Links; Last Reviewed.

**Conditional:** Taxonomy Relationship; Standard Relationship; Protocol Relationship; Output Boundary Relationship; Source Scope Notes; Related Reference Concepts; Publisher Candidate Notes; Candidate Route Notes; Public Claim Mapping Notes.

## H. Thin Content Failure Patterns

A future page fails if keyword-only, glossary-only without governance, generic AI commentary, repeated thesis, unsupported external claims, placeholder, future teaser, route filler, SEO-only, missing claim mapping, missing link logic, missing governance boundary, tool implication, subject guilt framing, or publisher-generated without approved candidate state.

## I. Minimum Substance Signals

Conceptual precision; boundary discipline; system relationship; traceable claims; source-scope clarity; semantic SEO discipline; non-sensational tone; future-proof route purpose; internal reference value; inspectable review posture.

## J. Content Quality Does Not Certify Truth

Passing this standard means the page has governed reference substance. It does not certify external factual truth, authenticity, legal status, forensic validity, deception, fraud, guilt, or event occurrence.
