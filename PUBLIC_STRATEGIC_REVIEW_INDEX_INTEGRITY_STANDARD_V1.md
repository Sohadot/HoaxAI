# Public Strategic Review Index Integrity Standard v1

Sprint 107 — Public Reference Strategic Review Index Integrity Audit v1  
**Decision:** DEC-125  
**Date:** 2026-06-27

## Strategic Review Index Integrity Standard Statement

Strategic-review-index integrity means the index organizes public inspection by strategic question without creating conclusions, scores, ratings, private access, transactions, reports, or operational claims.

## Strategic-review index route-count requirements

- Sitemap must remain at the current authorized public route count (78 after Sprint 106)
- Route registry must match sitemap count exactly
- public-file-registry must align with route HTML files if used
- Integrity audits must not add new public routes

## Strategic-review index component preservation requirements

Every strategic-review page must preserve Reference summary, Index purpose, Strategic review path, What this index supports, What this index does not claim, Reference Answer, Source Confidence, Cite This Reference, Retrieval Capsule, Boundary reminder, Non-transactional review boundary, and page-end reference navigation.

## Strategic-review index metadata requirements

Every strategic-review page must have exactly one H1, canonical URL, title element, meta description, Open Graph title, and Open Graph description.

## Strategic-review index link requirements

Every strategic-review page must link to `/strategic-review/`, at least three executive-overview, reviewer-packet, external-review, acquisition-readiness, entry-point, or narrative routes, at least five reference, utility, or pathway routes, and at least two sibling strategic-review routes. Internal links must resolve to registered public routes.

## Strategic-review index role clarity requirements

Every strategic-review page must clearly state it is a public strategic review index and route organizer by strategic question — not a scorecard, rating system, due-diligence room, pitch deck, sales page, private data room, downloadable report, transaction surface, legal representation, financial representation, verdict system, or operational review tool.

## Non-verdict requirements

Strategic-review pages must not imply automated authenticity labels, numeric certainty outputs, verdicts, uploads, detector behavior, or public automated reports.

## Non-transactional requirements

Strategic-review pages must not imply pricing statements, transaction pages, acquisition term documents, representative mandates, or legal or financial representation.

## Scorecard prohibition

Strategic-review pages must not function as scorecards. Safe negative language (`not a scorecard`) is required where role clarity is needed.

## Rating-system prohibition

Strategic-review pages must not function as rating systems. Safe negative language (`not a rating system`) is required where role clarity is needed.

## Due-diligence-room prohibition

Strategic-review pages must not claim or imply due-diligence-room access or private due-diligence room behavior.

## Pitch-deck prohibition

Strategic-review pages must not function as pitch decks. Safe negative language (`not a pitch deck`) is required where role clarity is needed.

## Sales-page prohibition

Strategic-review pages must not function as sales pages. Safe negative language (`not a sales page`) is required where role clarity is needed.

## Private data-room prohibition

Strategic-review pages must not claim or imply private data-room access or private due-diligence room behavior.

## Downloadable report prohibition

Strategic-review pages must not claim or imply downloadable report behavior.

## Forbidden regression patterns

Forbidden positive phrases in public HTML include scorecard, rating system, due diligence room, pitch deck, sales page, private data room, downloadable report, pricing statements, transaction pages, detector claims, upload behavior, and verdict language as capability claims.

## Future strategic-review-index audit rules

Future integrity audits must inspect the full public surface, preserve the Strategic Review Index Integrity Snapshot on the index hub, check scorecard, rating-system, and due-diligence-room drift, and record visible repairs before governance adoption.
