# Public Reference SEO Authority Map v1

## Purpose

Hoax.ai must grow as a reference asset, not as a page-volume project. Public SEO value comes from original, bounded, internally coherent reference substance that explains the category better than generic detector, scanner, or fake-real verdict pages.

This map defines the next authority layer for public reference production without creating any new public route, sitemap entry, navigation item, engine, classifier, upload workflow, scoring system, API, analytics, form, JavaScript behavior, monetization surface, or deployment change.

## Current Public Surface

The current public surface remains exactly nineteen URLs:

- home thesis surface
- category language layer
- twelve public reference pages
- Evidence Posture Standard v1
- Evidence Posture Protocol v1 Draft
- Evidence Field Interface Thesis with static embodiment

No sitemap expansion is authorized by this map. No page candidate listed here is public until it is separately registered, produced, internally linked, route-validated, sitemap-aligned, and passed through `validators/validate_all.py`.

## Strategic SEO Position

The strongest next additions are not generic articles. They are governed reference pages that deepen Hoax.ai's own category language:

- explain evidence posture concepts with original definitions
- connect concepts to the standard, protocol, and interface thesis
- preserve artifact-subject separation
- avoid deception verdicts and subject accusation
- avoid detector, scanner, upload, score, and fake-real promises
- make source, provenance, context, limitation, and output boundaries explicit
- create enough internal conceptual density that each page becomes a necessary reference node

## Authority Clusters

### Evidence Posture Core

This cluster protects the central category and should remain the anchor for all future reference production. Future pages must link back to Evidence Posture, Artifact-Subject Separation, Output Boundary, and the Evidence Posture Standard.

### Source And Provenance

This cluster should become the strongest non-operational alternative to generic verification content. It should define how source confidence, provenance gap, evidence chain, corroboration, lineage, and traceability relate without claiming to prove truth or identify deception.

### Context And Interpretation

This cluster should own the vocabulary around context loss, claim movement, interpretation risk, evidence limitation, and review restraint. It strengthens credibility by explaining what cannot be concluded.

### Standard Protocol Interface

This cluster connects public references to the Evidence Posture Standard, the protocol draft, and the Evidence Field Interface Thesis. It prevents future pages from drifting into isolated glossary entries.

## Candidate Pages

Candidate pages are planning objects only. They are not routes, not sitemap entries, not live navigation targets, and not authorization to publish.

1. `/reference/evidence-sufficiency/`
   - Defines when evidence is sufficient for a bounded posture but insufficient for a truth verdict.
2. `/reference/source-corroboration/`
   - Defines corroboration as source relationship discipline, not agreement-counting.
3. `/reference/context-window/`
   - Defines the minimum surrounding context needed before posture language can be responsible.
4. `/reference/provenance-chain/`
   - Extends provenance gap into a chain model without promising detection.
5. `/reference/media-lineage/`
   - Defines artifact lineage across origin, transformation, reposting, and presentation.
6. `/reference/synthetic-risk-signal/`
   - Explains signal language without presenting a detector score.
7. `/reference/artifact-integrity/`
   - Defines integrity as evidence condition, not authenticity certification.
8. `/reference/verification-burden/`
   - Defines where uncertainty should remain when proof is incomplete.
9. `/reference/claim-context/`
   - Defines how claims depend on surrounding context, source, and framing.
10. `/reference/evidence-threshold/`
    - Defines thresholds for posture language while preserving output restraint.
11. `/reference/interpretive-restraint/`
    - Defines institutional restraint when evidence posture is partial.
12. `/reference/posture-state/`
    - Defines state language across supported, limited, conflicted, and not assessable conditions.

## Production Rules

Every future public candidate must satisfy these rules before publication:

- no fake pages, placeholders, thin pages, or glossary-only entries
- no unsupported external factual claims
- no route without route registry approval
- no sitemap entry before route validation
- no orphaned page
- no page under the reference layer without internal links to the category language, standard, protocol where relevant, and at least two existing reference pages
- no engine, classifier, upload, scoring, API, analytics, form, JavaScript, monetization, detector, scanner, or fake-real verdict implication
- no subject accusation or deception proof language
- no publication unless `validators/validate_all.py` passes

Until then, this map is a planning and integrity artifact only.
