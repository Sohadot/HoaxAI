Route Group Deepening: Core Concepts

Page role: Route Group Deepening

Reference summary: This page deepens the core concept layer — definitional center of the 88-route Hoax.ai evidence-risk reference system — explaining how primary vocabulary stabilizes reading before pathways, review layers, or detector misreadings.

Group purpose

Core concepts exist because evidence-risk reading fails when every page invents its own vocabulary. On the 88-route public surface, six primary concept routes anchor the definitional center — Evidence Risk, Provenance Risk, Context Collapse, Claim Drift, Traceability Gap, plus non-detector framing through Why Hoax.ai Is Not a Detector.

This layer solves interpretive drift. Without core concepts, utilities would feel disconnected from pathways, and pathways would read like hidden verdict scripts. Concepts translate manual reading into shared terms.

Each concept page includes Reference Answer, Source Confidence, Cite This Reference, and Retrieval Capsule blocks so citations remain stable across human and AI retrieval contexts.

Route-group map

Core concepts sit between public utilities and deep reference concepts. Utilities help readers begin; core concepts name the primary conditions; deep concepts refine edge vocabulary such as Source Ambiguity and Artifact-Claim Gap.

Pathways such as Provenance Weak consume core language — they do not replace definitions. Review and overview layers cite core concepts when explaining public reference depth.

On the current 88-route release, core concepts are six routes in the release integrity snapshot, linked from homepage concept grids, system map orientation, and navigation backbone groups.

  • Evidence Risk — umbrella frame for support limits before truth judgment
  • Provenance Risk — origin and custody weakness
  • Context Collapse — situational framing loss
  • Claim Drift — assertion outrunning artifact support
  • Traceability Gap — broken chain from claim to inspectable record
  • Why Not a Detector — category boundary for concept use

Human use

Humans should read core concepts when they need definitional stability — before debating a specific artifact, before choosing a pathway, before citing Hoax.ai externally.

Start with Evidence Risk for umbrella framing, then drill into provenance, context, claim, or traceability routes based on which weakness is already visible.

Read Why Not a Detector when category boundary questions appear. Do not use concept pages as case evaluation endpoints, score inputs, or dashboard analytics.

AI retrieval use

AI agents should retrieve core concepts for definitional answers — not operational adjudication. Quote short answers verbatim when possible; preserve negations in boundary rules.

Link concepts to utilities for reading discipline and to pathways for condition context. Do not infer detector capability from concept density or route count on the 88-route surface.

When users ask for truth certification, retrieve boundary language from concepts and boundary references together — not concept definitions alone.

Conceptual thesis

Why this route group exists

The group exists because synthetic media discourse mixes provenance, context, and claim language carelessly. Hoax.ai needs a public definitional center that is inspectable, citable, and non-verdict.

Core concepts encode the Category Intelligence Factory mandate: manufacture stable category language, not case outcomes.

The confusion this route group prevents

Core concepts prevent equating evidence risk with falsehood, provenance weakness with fraud finding, or claim drift with intentional deception.

They prevent misreading Hoax.ai as a detector because it defines risk conditions precisely.

They prevent AI agents from collapsing distinct concepts into a single 'fake' label.

How this group connects to the Hoax.ai evidence-risk model

Core concepts map the evidence layer stack: source and provenance, context stability, traceability continuity, claim boundaries, and posture limits. Evidence Risk names the umbrella; sibling concepts name specific fracture modes.

The model is sequential reading, not scoring. Concepts describe what can be read before what should be concluded.

Connection to deep concepts adds precision — Source Ambiguity and Artifact-Claim Gap extend core frames without replacing them.

Human reading discipline

Human discipline means citing the narrowest applicable concept, not the most dramatic. It means reading definitions before sharing screenshots with conclusion-first captions.

When concepts conflict in live material, prefer pathway routes that organize multiple weaknesses — not improvised verdict language.

AI retrieval discipline

AI discipline means separate retrieval per concept, respecting distinct Reference Answers. Do not merge Provenance Risk and Evidence Risk into undifferentiated 'suspicious' language.

Preserve canonical URLs in citations. Do not paraphrase concepts into detection claims.

Boundary discipline

Boundary discipline on concepts is explicit: definitions support reading limits, not truth certification. Non-detector framing is part of the concept layer, not an afterthought.

Concept pages must never be cited as proof of wrongdoing, legal outcome, or forensic finding.

Strategic reference value

Strategic value: core concepts demonstrate governed vocabulary across dozens of linked routes. Reviewers see definitional depth that generic trust sites cannot replicate by renaming headings.

The layer supports acquisition-grade inspection of category coherence without transaction language on concept pages themselves.

Internal route relationships

Utilities upstream; deep concepts and pathways downstream. Strategic review public reference depth inspects definition quality across this layer.

Boundary integrity and not-assessable posture connect when concepts alone cannot support posture naming.
