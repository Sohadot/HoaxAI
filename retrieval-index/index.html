Hoax.ai Public Reference Retrieval Index

Page role: Public reference retrieval index organizing existing routes by retrieval intent

Reference summary: The retrieval index organizes public reference routes for citation-safe retrieval across the Hoax.ai 104-route surface. It is route orientation infrastructure for humans and AI agents. It is not a search interface, ranking surface, generated-answer layer, or case-assessment surface.

Purpose of the retrieval index

Hoax.ai has reached a depth where route quality and route discoverability must mature together. The purpose of this page is to make retrieval intent explicit. A reader should not have to guess where to begin for category framing, concept retrieval, condition retrieval, boundary retrieval, or strategic retrieval. This index gives a stable route orientation layer for those retrieval intents.

The index improves infrastructure usability without changing route claims. It does not reinterpret route content and does not compress route meaning into a generated abstraction. Instead, it points readers to governed routes already published on the public surface. Each link functions as companion reference routing for citation-safe use.

For AI systems, this page reduces retrieval drift by exposing intent-grouped references. The grouping helps agents choose route families that match the user question while preserving route-level boundaries. For humans, the same grouping reduces navigation fatigue and makes the public reference surface externally usable without turning it into an interactive product surface.

How to use this retrieval index

Start by identifying your retrieval intent, then open routes inside that intent group and cite route-level blocks directly. When a route includes Reference Answer, Source Confidence, Cite This Reference, or Retrieval Capsule sections, use those blocks before writing any synthesis. This preserves citation-safe retrieval and reduces interpretation drift.

Use this index for orientation, not automation. The index offers route orientation and companion references, not result selection. If multiple routes look relevant, choose the one whose page role and boundary language most closely match your reading question, then use additional companion routes for context.

When a question spans multiple intents, move in this order: category framing, concept retrieval, condition retrieval, crosswalk relation retrieval, then system-level navigation. This sequence helps maintain stable interpretation while avoiding category confusion between definitions, pathways, strategic framing, and boundary statements.

What this retrieval index is not

This retrieval index is not a search engine, not a search interface, not a recommendation engine, and not a generated-answer layer. It does not rank routes, score routes, or return best-result outputs. It does not summarize case material, verify material, detect material, or assess cases.

It is not a chatbot surface, not a decision-tree surface, not an operational review surface, and not a workflow surface. It contains no scripts, no forms, no uploads, no API behavior, and no dynamic filtering. Every route reference is static and manually inspectable.

It is also not a transaction surface. It is not a pricing statement, sales-page, consulting funnel, service funnel, legal representation surface, or financial representation surface. The index remains a non-transactional reference navigation layer.

Human retrieval use

Humans can use this page to rapidly move from question type to route family. A policy reader can start with boundary language; a research reader can start with concepts and conditions; a strategic reader can start with review and overview layers. This intent-first retrieval reduces route confusion and increases citation consistency.

Human-readable route orientation matters because the public surface is now broad. A static intent index improves usability while preserving transparency: every link is explicit, every route is inspectable, and no hidden logic chooses references for the reader.

AI retrieval use

AI systems can use this index as a retrieval-safe route selector by question intent. The index supports grounded retrieval by surfacing companion references before answer synthesis. It helps models preserve boundary-safe retrieval and avoid fabricated route relations.

When a user prompt is ambiguous, agents should retrieve from this index first, choose the intent group that best fits the question, and then quote route-level blocks from target pages. This keeps output traceable to canonical references rather than inferred summaries.
