# Public System Map Standard v1

Sprint 108 — Public Reference System Map Surface v1  
**Decision:** DEC-126  
**Date:** 2026-06-27

## System Map Standard Statement

System map pages organize public structural inspection by layer without creating scorecards, rating systems, dashboards, conclusions, transaction claims, private access, reports, or operational claims.

## Required page structure

Reference summary, Map purpose, System map path, What this map supports, What this map does not claim, Reference Answer, Source Confidence, Cite This Reference, Retrieval Capsule, Related Routes, page-end navigation, Boundary reminder, Non-transactional review boundary.

## Required map components

Stable anchors: reference-summary, map-purpose, system-map-path, reference-answer, source-confidence, cite-this-reference, retrieval-capsule, boundary.

## System map path requirements

Five to seven linked steps through existing Hoax.ai routes per page.

## Layer map requirements

Hub links all child maps; child maps link hub and siblings; homepage links all five routes.

## Human readability requirements

At least 850 visible words per new route; citation-safe wording; one H1 per page.

## AI readability requirements

Reference Answer, Source Confidence, Cite This Reference, Retrieval Capsule on every page.

## Citation/retrieval requirements

Canonical URLs, meta descriptions, Open Graph title and description on every page.

## Internal link requirements

Link to `/system-map/`, at least three strategic layer routes, at least five reference or utility routes, at least two sibling system-map routes.

## Non-verdict requirements

No detector claims, verdict language, upload, or automated report promises.

## Non-transactional requirements

No pricing, transaction, legal, or financial representation.

## Scorecard prohibition

Safe negative language required; no scorecard framing.

## Rating-system prohibition

Safe negative language required; no rating-system framing.

## Dashboard prohibition

Safe negative language required; no dashboard, analytics panel, or interactive rating widget framing.

## Pitch-deck prohibition

Safe negative language required; no pitch-deck framing.

## Sales-page prohibition

Safe negative language required; no sales-page framing.

## Private data-room prohibition

Safe negative language required; no private data-room access.

## Downloadable-report prohibition

Safe negative language required; no downloadable-report behavior.

## Stale route-count prohibition

Maps must use the current 83-route public surface count; stale 78-route language is forbidden.

## Governance alignment

DEC-126, PUB-GATE-0102, CLAIM-0109, and `validators/validate_public_reference_system_map_surface_v1.py` enforce this standard.
