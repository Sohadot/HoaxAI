# Hoax.ai Interface Thesis

## Governing Principle

The interface must not decorate Hoax.ai.

It must embody the thesis.

Hoax.ai is about evidence posture, synthetic risk, source confidence, provenance gaps, contextual uncertainty, and deception patterns.

The interface must help users understand that trust is not a single binary answer.

Trust is a layered structure.

## Interface Thesis

**The interface must show evidence posture, not perform truth judgment.**

Hoax.ai must not look like a dramatic fake/real detector, a gaming interface, or a generic cyber dashboard.

It should not visually imply that the system has final authority over truth.

The visual experience should communicate layered examination, bounded confidence, and structured uncertainty.

## Visual Model

The interface should express evidence as layers:

- claim layer;
- source layer;
- provenance layer;
- media layer;
- context layer;
- identity layer;
- pattern layer;
- posture layer.

Users should feel that they are examining the structure of evidence, not receiving a magic answer.

## Motion and Interaction

Motion must have meaning.

Allowed motion:

- revealing evidence layers;
- showing uncertainty states;
- moving from claim to source to context;
- demonstrating provenance gaps;
- showing posture transitions.

Prohibited motion:

- decorative animations with no conceptual purpose;
- aggressive scanning effects that imply absolute detection;
- fake AI loading screens;
- gaming-style threat meters;
- sensational red-alert visuals;
- or interface patterns that create fear without explanation.

## Accessibility and Performance

The interface must be:

- accessible;
- mobile-stable;
- SEO-safe;
- fast;
- readable without JavaScript;
- understandable without animation;
- and functional without advanced visual effects.

WebGL or 3D may only be used later if it explains evidence structure and does not damage performance, accessibility, trust, or search visibility.

## Design Hierarchy

Concept first.  
Performance second.  
Beauty third.

Beauty is valuable only when it clarifies the thesis.

## Interface Test

Every interface element must pass these questions:

1. Does it explain evidence posture?
2. Does it support the thesis?
3. Does it avoid false authority?
4. Does it remain understandable without motion?
5. Does it preserve accessibility?
6. Does it protect SEO?
7. Does it load quickly?
8. Does it increase trust?
9. Does it avoid childish or gaming aesthetics?
10. Does it make Hoax.ai feel like category infrastructure?

If not, it should not be used.

## Governing Sentence

**Show the evidence layers. Do not simulate the verdict.**
