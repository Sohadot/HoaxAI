# Public Reference Strategic Review Index v1

Sprint 106 — Public Reference Strategic Review Index v1  
**Decision:** DEC-124  
**Date:** 2026-06-26

## Strategic Review Index Statement

Hoax.ai adopts a public strategic review index that organizes inspection by strategic question: category position, public reference depth, retrieval and citation readiness, and boundary/readiness discipline — without conclusions, scores, transaction claims, or private access.

## Why the strategic review index exists after Sprint 105

After Sprint 105 protected the executive overview surface, serious reviewers and AI agents needed a route index organized by strategic question rather than random browsing across the growing public surface.

## Routes created

| Route | Role |
|-------|------|
| `/strategic-review/` | Strategic review index hub |
| `/strategic-review/category-position/` | Category position review index |
| `/strategic-review/public-reference-depth/` | Public reference depth review index |
| `/strategic-review/retrieval-and-citation/` | Retrieval and citation review index |
| `/strategic-review/boundary-and-readiness/` | Boundary and readiness review index |

## Category position review role

Indexes thesis, narrative, non-detector positioning, and strategic readiness routes for category diligence.

## Public reference depth review role

Indexes utilities, concepts, pathways, and surface-index routes for depth and coherence inspection.

## Retrieval and citation review role

Indexes citation maps, AI review guides, and retrieval-hardened reference blocks — not API behavior.

## Boundary and readiness review role

Indexes boundary discipline and public inspectability routes across review surfaces.

## Human reviewer usefulness

Reviewers choose inspection paths by strategic question without sitemap-only browsing.

## AI retrieval usefulness

Agents retrieve stable review paths, Reference Answers, and boundary reminders without inferring operational authority.

## Non-verdict boundary

No automated authenticity labels, verdicts, uploads, or public reports.

## Non-transactional boundary

No pricing, transaction, pitch-deck, sales-page, private data-room, or downloadable-report behavior.

## Why this is visible production, not abstract governance

Five live HTML routes, homepage navigation, and release snapshot updates constitute visible production contact.

## Future strategic-review-index candidates

Integrity audit snapshot on hub; additional question-specific index routes only through governed sprints.
