# Public Route Group Deepening Standard v1

Sprint 112 — Public Reference Route Group Deepening v1  
**Decision:** DEC-130

## Route Group Deepening Standard Statement

Route-group deepening pages explain major public reference layers as usable human and AI-retrieval surfaces without creating dashboards, graph tools, scorecards, rating systems, due-diligence rooms, private data-room access, downloadable reports, pitch decks, sales pages, upload behavior, scoring, verdicts, detector behavior, or automated reports.

## Required page structure

1. Reference summary  
2. Group purpose  
3. Route-group map  
4. Human use  
5. AI retrieval use  
6. What this group supports  
7. What this group does not claim  
8. Reference Answer  
9. Source Confidence  
10. Cite This Reference  
11. Retrieval Capsule  
12. Page-end reference navigation  
13. Boundary reminder  
14. Non-transactional review boundary  

Additional depth sections: conceptual thesis, why this route group exists, confusion prevented, evidence-risk model connection, human reading discipline, AI retrieval discipline, boundary discipline, strategic reference value, internal route relationships, what would weaken this group, safe citation summary.

## Minimum depth requirement

At least 1,000 visible words per route-group deepening page (Sprint 112 production target: 1,400+ words).

## Required reference components

reference_summary, group_purpose, route_group_map, human_use, ai_retrieval_use, reference_answer, source_confidence, cite_this_reference, retrieval_capsule, page_end_reference_navigation, boundary_reminder, non_transactional_review_boundary.

## Route-group map requirements

Each page must link representative routes within its group and explain layer placement on the 88-route surface.

## Human use requirements

Explain ordered reading discipline and what the group must not be used to infer.

## AI retrieval use requirements

Explain safe citation, Retrieval Capsule use, and boundary preservation — not operational authority.

## Internal link requirements

Each page links to homepage, `/system-map/`, at least two sibling route-group deepening pages, at least six existing reference routes, and at least two review/overview/system-map routes.

## Citation/retrieval requirements

Stable anchor IDs, Reference Answer blocks, Source Confidence blocks, Cite This Reference blocks, and Retrieval Capsules on every page.

## Non-verdict requirements

No case evaluation, conclusion, score, rating, detector output, or automated report promise.

## Non-transactional requirements

No pricing, transaction, acquisition term, representative mandate, legal representation, or financial representation.

## Dashboard prohibition

No interactive dashboards, analytics panels, or rating widgets.

## Graph-tool prohibition

No graph visualization tools or interactive network maps.

## Scorecard prohibition

No scorecards or graded surface summaries.

## Rating-system prohibition

No rating systems or numeric certainty outputs.

## Due-diligence-room prohibition

No private due-diligence room framing.

## Pitch-deck prohibition

No pitch-deck behavior or downloadable deck claims.

## Sales-page prohibition

No sales-page behavior or purchase framing.

## Private data-room prohibition

No private data-room access claims.

## Downloadable report prohibition

No downloadable report behavior.

## Forbidden regression patterns

Upload fields, forms, JavaScript interaction, public API behavior, chatbot/generator behavior, real-world case evaluation, forbidden positive promotional phrases in public HTML.

## Thin-page prevention rules

No generic filler, no keyword stuffing, no pages that could belong to any generic trust website; each page must have a unique conceptual thesis native to Hoax.ai.
